# Keyboard UX Specification

모든 대시보드 액션은 키보드만으로 접근 가능해야 한다.
새 기능을 추가할 때 이 문서의 레이어/존 모델에 맞춰 키보드 접근 경로를 함께 설계할 것.

---

## 1. 레이어 모델

키보드 입력은 항상 하나의 **활성 레이어**에서만 처리된다. 상위 레이어가 열리면 하위 레이어의 키 입력을 가로챈다.

```
우선순위 (높은 순):

4. Text Input    — input/textarea/contenteditable에 포커스
3. Dialog        — CLI 시작 다이얼로그
2. Modal         — 문서 모달
1. Terminal      — 터미널 패널 (iframe 포커스)
0. Board         — 기본 레이어 (컬럼 네비게이션)
```

**레이어 전환 규칙:**
- 상위 레이어가 열리면 → 자동으로 해당 레이어가 활성화
- Escape → 현재 레이어를 닫고 이전 레이어로 복귀
- Text Input 레이어에서 Escape → 블러 후 이전 레이어로 복귀

---

## 2. Board 레이어 — 존 네비게이션

Board는 가로로 나열된 **존(zone)** 단위로 탐색한다.

```
[ 프로젝트A ] [ 프로젝트B ] [ 프로젝트C ] [ 자유 공간 ]
   zone 0       zone 1       zone 2       zone 3
```

- 프로젝트 존: 프로젝트 컬럼 (`.col`)
- 자유 공간 존: 컬럼 오른쪽 나머지 영역 (포스트잇 노트)

### 네비게이션 키

| 키 | 동작 |
|----|------|
| `←` `→` | 존 간 이동 (좌우) |
| `↑` `↓` | 현재 존 내 포커스 아이템 이동 (상하) |
| `Enter` / `Space` | 포커스된 아이템 활성화 (클릭 효과) |
| `Escape` | 포커스 해제 (아이템 → 존 → 없음) |

### 포커스 계층

```
없음 → 존 포커스 (컬럼 하이라이트) → 아이템 포커스 (개별 요소 하이라이트)
```

- 아무 포커스 없는 상태에서 방향키 → 첫 번째 존에 존 포커스
- 존 포커스 상태에서 `↑`/`↓` → 첫 아이템으로 아이템 포커스 진입
- 아이템 포커스 상태에서 `←`/`→` → 인접 존으로 이동 (아이템 인덱스 유지 시도)
- 아이템 포커스 상태에서 `Escape` → 존 포커스로 복귀
- 존 포커스 상태에서 `Escape` → 포커스 해제

### 프로젝트 존의 포커스 가능 아이템 (위→아래 순서)

| 순서 | 요소 | 활성화 동작 |
|------|------|-------------|
| 0 | col-label (헤더) | 디테일 토글 (확장/축소) |
| 1 | docs-btn | 문서 모달 열기 → Modal 레이어 |
| 2 | term-card | 터미널 열기 → Terminal 레이어 (off면 Dialog 레이어) |
| 3+ | handoff cards | 카드 확장/축소 |

### 자유 공간 존의 포커스 가능 아이템

| 순서 | 요소 | 활성화 동작 |
|------|------|-------------|
| 0~N | 포스트잇 노트 (Y좌표 순) | 노트 확장 → Text Input 레이어 |

---

## 3. Modal 레이어 — 문서 모달

문서 모달이 열린 상태의 키보드 동작.

| 키 | 동작 |
|----|------|
| `Escape` | 모달 닫기 → Board 레이어 복귀 |
| `←` `→` | 탭 전환 (Roadmap / Readme / Architecture) |
| `↑` `↓` | 문서 내용 스크롤 |

---

## 4. Dialog 레이어 — CLI 시작 다이얼로그

CLI 시작 다이얼로그가 열린 상태의 키보드 동작.

| 키 | 동작 |
|----|------|
| `Escape` | 다이얼로그 닫기 → Board 레이어 복귀 |
| `↑` `↓` | CLI 명령어 목록 내 선택 이동 |
| `Enter` | 선택된 명령어로 CLI 시작 |
| (타이핑) | 커스텀 입력란 자동 포커스 |

---

## 5. Terminal 레이어 — 터미널 패널

터미널 패널이 열린 상태. iframe 내부가 키 입력을 받으므로 iframe 외부 영역 클릭이나 단축키로 제어.

| 키 | 동작 |
|----|------|
| `Escape` | 터미널 닫기 → Board 레이어 복귀 (iframe에 포커스가 없을 때) |

**참고:** iframe에 포커스가 있으면 키 입력이 ttyd로 전달된다. Board로 돌아오려면 iframe 바깥을 클릭하거나, 향후 전용 단축키(예: `Ctrl+Escape`)를 추가할 수 있다.

---

## 6. 글로벌 단축키

레이어와 무관하게 항상 작동하는 키. Text Input 레이어에서는 비활성.

| 키 | 동작 | 비고 |
|----|------|------|
| `Shift+Space` | 활성 세션(running/idle) 간 터미널 순환 스위칭 | 세션이 없으면 무동작. 터미널이 닫혀있어도 열면서 전환 |
| `Ctrl+M` | 노트 추가 | |

---

## 7. 시각 피드백

| 상태 | 시각 표현 |
|------|-----------|
| 존 포커스 | 컬럼 좌측에 세로 액센트 라인 (또는 배경 tint) |
| 아이템 포커스 | 해당 요소에 아웃라인 또는 배경 하이라이트 |
| 포커스 없음 | 별도 표시 없음 |

---

## 8. 구현 가이드

### 활성 레이어 판별 (우선순위 순)

```
1. document.activeElement가 input/textarea/contenteditable → Text Input
2. CLI 다이얼로그가 visible → Dialog
3. 문서 모달이 visible → Modal
4. 터미널 패널이 open + iframe 포커스 → Terminal
5. 그 외 → Board
```

### 상태 변수

```javascript
let focusZone = -1;   // -1: 없음, 0~N-1: 프로젝트, N: 자유 공간
let focusItem = -1;   // -1: 존 포커스만, 0+: 아이템 인덱스
```

### 새 기능 추가 시 체크리스트

1. 이 기능은 어떤 레이어에 속하는가?
2. 해당 레이어에서 키보드로 어떻게 진입하는가?
3. 포커스 가능 아이템이 추가되는가? → 아이템 순서 테이블 업데이트
4. 새 레이어가 필요한가? → 레이어 우선순위 테이블 업데이트
5. Escape로 이전 레이어 복귀가 되는가?
