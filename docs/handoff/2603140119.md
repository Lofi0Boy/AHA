## Architecture Decisions Made This Session

- **All Claude instances use Claude Code CLI** — not API — so every agent has full local file control
- **One persistent Claude Code session per project** — spawned once, reused via `--resume`; context resets naturally on compaction (handoff written → session restart)
- **Python daemon is the process manager only** — holds state, manages lifecycles; intelligence lives in Claude Code sessions
- **CLI is the single I/O source of truth** — dashboard renders it, Telegram bridges it; no separate logic per channel
