**팝아웃 ↔ 메인 페이지 통신을 BroadcastChannel로 전면 교체**
- 기존 `window.opener.postMessage` → `BroadcastChannel('mpm-terminal')`
- `window.opener`는 메인 페이지 새로고침 시 null이 되어 모든 통신 실패했음
- BroadcastChannel은 같은 origin의 모든 탭/창이 공유 — 새로고침과 무관
- 팝아웃 추적: `popOutWindows` (window 참조 dict) → `activePopouts` (프로젝트 이름 Set)
- 팝아웃 등록/해제: `mpm-popout-register`, `mpm-popout-closed` 메시지
- 메인 페이지 새로고침 시 `mpm-ping` → 팝아웃들이 재등록

- 팝아웃: `term.textarea.addEventListener('focus')` → `mpm-focus` broadcast

**서버 재시작 시 세션 보존 (60초 grace period)**
- systemd `KillMode=process` — 메인 Python만 죽이고 tmux/ttyd는 살림
- 셧다운 시 `data/.server_shutdown_at` 타임스탬프 마커 기록
- 시작 시 마커 확인: 60초 미만 → `reconnect_ttyd()`로 ttyd 재연결, 60초 초과 → `cleanup_all()`
- 마커 없어도 항상 `reconnect_ttyd()` 실행 (기존 tmux 세션에 ttyd 붙이기)

**ttyd 포트 안정화 + orphan 정리**

- 3/16 이후 쌓인 좀비 ttyd 16개가 WebSocket 연결을 가로채던 문제 해결

**Flask 템플릿/브라우저 캐싱 비활성화**
- `TEMPLATES_AUTO_RELOAD = True` — 서버 재시작 없이 템플릿 변경 반영
- `@app.after_request` — `Cache-Control: no-store` 헤더로 브라우저 캐싱 비활성화
